Overview

select_nodes() is an advanced node selection function that improves upon filter_nodes() with lazy centrality computation, multiple selection modes, and global context variables for structural awareness.

Function Signature

select_nodes(
  x,
  ...,                           # dplyr-style filter expressions
  name = NULL,                   # Character vector of node names
  index = NULL,                  # Integer vector of node indices
  top = NULL,                    # Select top N nodes
  by = "degree",                 # Centrality measure for top selection
  neighbors_of = NULL,           # Select neighbors of these nodes
  order = 1L,                    # Neighborhood order (1 = direct)
  component = NULL,              # "largest", integer, or node name
  .keep_edges = c("internal", "none"),
  keep_format = FALSE,
  directed = NULL
)
Key Features
  • Lazy computation - Only computes centrality measures actually referenced
  • Multiple selection modes - By name, index, top N, neighborhood, component
  • Global context - Access component membership, articulation points, k-core
  • AND logic - All modes combine with intersection (like tidygraph)
  • 4x faster than filter_nodes() for simple queries

select_nodes() vs filter_nodes()

Feature filter_nodes() select_nodes()
Expression filtering Yes Yes
Centrality computation All measures upfront Lazy (only needed)
Select by name Via expression Direct parameter
Select by index Via expression Direct parameter
Top N selection Manual ranking Built-in
Neighborhood selection Not available Built-in
Component selection Not available Built-in
Global context vars Not available 7 variables
Negative weight handling May error Warns + NA

When to Use Which

Use filter_nodes() when:

  • Simple centrality filtering
  • You need all centrality measures anyway
  • Backward compatibility needed

Use select_nodes() when:

  • Performance matters (large networks)
  • Selecting by name, index, or top N
  • Neighborhood or component extraction
  • Need global context (articulation points, etc.)

Selection Modes

Mode Parameter Example
Expression ... select_nodes(x, degree >= 3)
By name name select_nodes(x, name = c("A", "B"))
By index index select_nodes(x, index = c(1, 3, 5))
Top N top, by select_nodes(x, top = 10, by = "pagerank")
Neighborhood neighbors_of, order select_nodes(x, neighbors_of = "A", order = 2)
Component component select_nodes(x, component = "largest")

Mode 1: Expression Filtering (Lazy)

# Only computes degree - fast!
select_nodes(net, degree >= 3)

# Only computes pagerank
select_nodes(net, pagerank > 0.1)

# Computes both degree and betweenness
select_nodes(net, degree >= 3 & betweenness > 0)

Mode 2: By Name

# Select specific nodes by label
select_nodes(net, name = c("Alice", "Bob", "Charlie"))

# Equivalent to (but faster than):
filter_nodes(net, label %in% c("Alice", "Bob", "Charlie"))

Mode 3: By Index

# Select nodes 1, 3, and 5
select_nodes(net, index = c(1, 3, 5))

# First 10 nodes
select_nodes(net, index = 1:10)

Mode 4: Top N by Centrality

# Top 10 by degree (default)
select_nodes(net, top = 10)

# Top 5 by PageRank
select_nodes(net, top = 5, by = "pagerank")

# Top 10 by betweenness
select_nodes(net, top = 10, by = "betweenness")

# Available measures for 'by':
# degree, indegree, outdegree, strength, instrength, outstrength,
# betweenness, closeness, eigenvector, pagerank, hub, authority, coreness

Mode 5: Neighborhood (Ego Network)

# Direct neighbors of node "A" (includes A itself)
select_nodes(net, neighbors_of = "A")

# 2-hop neighborhood
select_nodes(net, neighbors_of = "A", order = 2)

# Neighborhood of multiple nodes
select_nodes(net, neighbors_of = c("A", "B"), order = 1)

# By index instead of name
select_nodes(net, neighbors_of = 1, order = 2)

Mode 6: Component Selection

# Largest connected component
select_nodes(net, component = "largest")

# Component containing node "A"
select_nodes(net, component = "A")

# Component by ID (from igraph::components)
select_nodes(net, component = 2)

Global Context Variables

These variables provide structural information about each node's position in the network. They are computed lazily - only when referenced in an expression.

Variable Type Description
component Integer Component membership ID (from igraph::components)
component_size Integer Number of nodes in the node's component
is_largest_component Logical TRUE if node is in the largest connected component
neighborhood_size Integer Number of direct neighbors (same as degree)
k_core Integer K-core membership (coreness value)
is_articulation Logical TRUE if node is an articulation point (cut vertex)
is_bridge_endpoint Logical TRUE if node is incident to a bridge edge

Examples

# Nodes in the largest connected component
select_nodes(net, is_largest_component)

# Articulation points (critical nodes)
select_nodes(net, is_articulation)

# Nodes at bridge endpoints
select_nodes(net, is_bridge_endpoint)

# Nodes in 3-core or higher (densely connected)
select_nodes(net, k_core >= 3)

# Large components only
select_nodes(net, component_size >= 10)

# Combine with centrality
select_nodes(net, is_articulation & degree >= 5)

# High-degree nodes in the largest component
select_nodes(net, is_largest_component & degree >= 3)
What are Articulation Points?

An articulation point (or cut vertex) is a node whose removal would disconnect the graph. These are critical nodes for network connectivity.

What are Bridge Edges?

A bridge is an edge whose removal would disconnect the graph. is_bridge_endpoint identifies nodes at the ends of these critical edges.

Convenience Functions

Three helper functions for common selection patterns:

select_neighbors()

select_neighbors(
  x,
  of,
  order = 1,
  ...
)

Extract ego network around focal nodes.

select_component()

select_component(
  x,
  which = "largest",
  ...
)

Extract a connected component.

select_top()

select_top(
  x,
  n,
  by = "degree",
  ...
)

Select top N nodes by centrality.

Examples

# Ego network of "Hub" with 2-hop radius
select_neighbors(net, of = "Hub", order = 2)

# Largest connected component
select_component(net, which = "largest")

# Component containing "A"
select_component(net, which = "A")

# Top 10 by eigenvector centrality
select_top(net, n = 10, by = "eigenvector")

# Top 5 by PageRank, then filter by degree
select_top(net, n = 5, by = "pagerank", degree >= 3)

Combining Modes (AND Logic)

All selection modes combine with AND logic (intersection). A node must satisfy ALL specified criteria to be selected. 

# Top 10 by PageRank WITHIN the largest component
select_nodes(net, component = "largest", top = 10, by = "pagerank")

# Neighbors of "A" that also have high degree
select_nodes(net, neighbors_of = "A", degree >= 5)

# Top 5 among specific named nodes
select_nodes(net, name = c("A", "B", "C", "D", "E"), top = 3)

# Articulation points in the largest component
select_nodes(net, component = "largest", is_articulation)

# High-degree nodes in 2-hop neighborhood of "Hub"
select_nodes(net, neighbors_of = "Hub", order = 2, degree >= 4)
Order of Operations

Modes are applied in this order: name, index, component, neighbors_of, top, expressions. The top selection is applied to the nodes remaining after other filters.

Performance

select_nodes() uses lazy computation - it only calculates centrality measures that are actually referenced in your query. This provides significant speedup for simple queries.

Benchmark: 500-node Network

Filtering by degree >= 10 (3 iterations):

filter_nodes() - computes ALL centralities:

0.80s

select_nodes() - computes only degree:

0.20s (4x faster)
Performance Tips
  • Use select_nodes() for large networks (500+ nodes)
  • Prefer simple centrality measures (degree, strength) when possible
  • Use top parameter instead of sorting and filtering manually
  • Use name or index parameters for direct selection

Complete Examples

Example 1: Network Core Extraction

# Extract the densely connected core of a network
core <- net |>
  select_nodes(component = "largest") |>
  select_nodes(k_core >= 3) |>
  select_nodes(degree >= 5)

splot(core)

Example 2: Ego Network Analysis

# Analyze the neighborhood around key nodes
ego_net <- select_neighbors(net, of = c("CEO", "CTO"), order = 2)

# Find brokers (articulation points) in the ego network
brokers <- select_nodes(ego_net, is_articulation)

splot(ego_net)
splot(brokers)

Example 3: Influence Analysis

# Top influencers by different measures
top_degree <- select_top(net, n = 10, by = "degree")
top_pagerank <- select_top(net, n = 10, by = "pagerank")
top_between <- select_top(net, n = 10, by = "betweenness")

# Compare the sets
cat("Top by degree:", get_labels(top_degree), "\n")
cat("Top by PageRank:", get_labels(top_pagerank), "\n")
cat("Top by betweenness:", get_labels(top_between), "\n")

Example 4: Critical Node Identification

# Find critical nodes for network resilience
critical <- select_nodes(net,
  is_articulation | is_bridge_endpoint,
  degree >= 3
)

# Visualize with highlighting
splot(net)
splot(critical)

Example 5: Disconnected Network Analysis

# Analyze each component separately
largest <- select_component(net, which = "largest")
second <- select_component(net, which = 2)

# Compare structure
cat("Largest component:", n_nodes(largest), "nodes\n")
cat("Second component:", n_nodes(second), "nodes\n")

Example 6: TNA Workflow

# Extract significant states from TNA model
tna_net <- as_cograph(tna_model)

# High-traffic states with significant initial probability
key_states <- select_nodes(tna_net,
  inits > 0.05,
  strength > 0.3
)

splot(key_states)

Handling Negative Weights

Some centrality measures (betweenness, closeness) cannot be computed with negative edge weights. select_nodes() handles this gracefully: 

# Network with negative weights
mat <- matrix(c( 0,  0.5, -0.3,
                0.5,  0,  0.4,
               -0.3, 0.4,  0), 3, 3)

# This will warn and return NA for betweenness
select_nodes(mat, betweenness > 0)
#> Warning: Betweenness cannot be computed with negative weights. Returning NA.

# Degree works fine (doesn't use weights for distance)
select_nodes(mat, degree >= 2)
#> Works normally
Affected Measures

With negative weights, betweenness and closeness will return NA with a warning. Other measures work normally.
